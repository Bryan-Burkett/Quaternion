This library comes with an optional file called "qreader.rkt". It allows for the interpreter to read quaternions in the form a+bi+cj+dk and convert them to the form (quaternion a b c d). To use this feature type (require "qreader.rkt") in the definitions window and run your program. In the interpreter type (start). Then you can use the interpreter as you normally would, typing quaternions in their standard infix notation.

1 Basic User Guide

1.1 What a Quaternion is and how to create a Quaternion in Racket

Quaternions are complex number of the form w + xi + yj + zk, where w, x, y, z are real numbers and i, j, k are imaginary units that satisfy certain conditions. To use Quaternioin.rkt, at the top of your file type (require "Quaternion.rkt"). This will allow you to use the procedures that were created in "Quaternion.rkt".

To create a Quaternion in racket, type
>(q 1 2 3 4)
(quaternion 1 2 3 4)
or
>(quaternion 1 2 3 4)
(quaternion 1 2 3 4)
They are two different ways of creating a quaternion. Both are interchangeable.

The "(quaternion 1 2 3 4)" that was returned represents 1 + 2i + 3j + 4k in the standard form.

Real part of a Quaternion: Since Quaternions are complex numbers meaning they have a real and imaginary part we created a function that will return you the real part of the Quaternion that you input.

To find the real part of a Quaternion in racket, type
> (scalar-part (q 1 2 3 4))
1
The "1" that was returned represents the real part of the Quaternion.

Vector part of a Quaternion: Since Quaternions are complex numbers meaning they have a real and imaginary part we created a function that will return you the vector part of the Quaternion that you input.

To find the vector part of a Quaternion in racket, type
> (vector-part (q 1 2 3 4))
(quaternion 0 2 3 4)
The "(quaternion 0 2 3 4)" that was returned represents the vector part of the Quaternion which would be 2i + 3j + 4k in standard form.

1.2 How To Use The Basic Operators For Quaternions

How to add Quaternions: To add two quaternions you have to add each component separately. For example if we had the two quaternions 2 + 3i + 4j + 5k and 1 + 2i + 3j + 4k to add them we would just simply add each individual component. For this example we would add 2 + 1 = 3, 3i + 2i = 5i, 4j + 3j = 7j and lastly 5k + 4k = 9k which gives us the quaternion 3 + 5i + 7k + 9k.

To add two Quaternions in racket, type
> (+ (q 2 3 4 5) (q 1 2 3 4))
(quaternion 3 5 7 9)
The "(quaternion 3 5 7 9)" that was returned is the result of adding each component of the two quaternions.

How to subtract Quaternions: To add two quaternions you have to add each component separately. For example if we had the two quaternions 2 + 3i + 4j + 5k and 1 + 2i + 3j + 4k to subtract them we would just simply subtract each individual component. For this example we would subtract 2 - 1 = 1, 3i - 2i = 1i, 4j - 3j = 1j and lastly 5k - 4k = 1k which gives us the quaternion 1 + 1i + 1k + 1k.

To subtract two Quaternions in racket, type
> (- (q 2 3 4 5) (q 1 2 3 4))
(quaternion 1 1 1 1)
The "(quaternion 3 5 7 9)" that was returned is the result of subtracting each component of the two quaternions.

How To Multiply Quaternions: To multiply two quaternions we have to take the cross product of the two quaternions. For example if we had the two quaternions 2 + 3i + 4j + 5k and 1 + 2i + 3j + 4k to multiply them we would do the following. To get the real part of the new quaternion let’s call it t0 we would do the following t0 = (2*1 - 3*2 - 4*3 - 5*4) = -36. To calculate the i component, let’s call it t1, we would do the following t1 = (1*3 + 2*2 - 3*5 + 4*4) = 8i. To calculate the j component, let’s call it t2, we would do the following t2 = (1*4 + 2*5 + 3*2 - 4*3) = 8j. To calculate the k component, let’s call it t3, we would do the following t3 = (1*5 - 2*4 + 3*3 + 4*2) = 14k. Combine the components back together and we get the quaternions -36 + 8i + 8j + 14k.

To multiply two Quaternions in racket, type
> (* (q 2 3 4 5) (q 1 2 3 4))
(quaternion -36 8 8 14)
The "(quaternion -36 8 8 14)" that was returned is the result of multiplying the two quaternions together.

How To Divide Quaternions: To divide two quaternions we have to again break the problem into smaller parts and find each component separately. For example if we had the two quaternions 2 + 3i + 4j + 5k and 1 + 2i + 3j + 4k to divide them we would do the following. To get the real part of the new quaternion let’s call it t0 we would do the following, t0 = (2*1 + 3*2 + 4*3 + 5*4)/(1^2 + 2^2 + 3^2 + 4^2) = 1.3333. To calculate the i component, let’s call it t1, we would do the following t1 = (1*3 - 2*2 + 3*5 - 4*3)/(1^2 + 2^2 + 3^2 + 4^2) = -.066666i. To calculate the j component, let’s call it t2, we would do the following t2 = (1*4 + 2*5 - 3*2 - 4*3)/(1^2 + 2^2 + 3^2 + 4^2) = 5.5e-17j which is pretty closer to zero. To calculate the k component, let’s call it t3, we would do the following t3 = (1*5 + 2*4 - 3*3 - 4*2)/(1^2 + 2^2 + 3^2 + 4^2) = -.133333k. Combine the components back together and we get the quaternions 1.333 - .0666i + 0j - .13333k.

To divide two Quaternions in racket, type
> (/ (q 2 3 4 5) (q 1 2 3 4))
(quaternion 1.3333333333333335 -0.06666666666666665 5.551115123125783e-17 -0.13333333333333333)
The "(quaternion 1.3333333333333335 -0.06666666666666665 5.551115123125783e-17 -0.13333333333333333)" that was returned is the result of dividing the two quaternions.

1.3 How To Use Trigonometric Functions For Quaternions

How To Find The Sine Of A Quaternion To calculate the sine of a quaternion we used the following formula sin(1 + 2i + 3j + 4k) = (e^((1 + 2i + 3j + 4k)*((2i + 3j + 4k)/(||2i + 3j + 4k||))) + e^-((1 + 2i + 3j + 4k)*((2i + 3j + 4k)/(||2i + 3j + 4k||)))) / 2.

To find the sine of a Quaternion in racket, type
> (sin (q 1 2 3 4))
(quaternion 91.78371578403477 21.88648685302919 32.82973027954379 43.77297370605838)
The "(quaternion 91.78371578403477 21.88648685302919 32.82973027954379 43.77297370605838)" that was returned is the sine of the input quaternion.

How To Find The Cosine Of A Quaternion To calculate the cosine of a quaternion we used the following formula cos(1 + 2i + 3j + 4k) = (e^((1 + 2i + 3j + 4k)*((2i + 3j + 4k)/(||2i + 3j + 4k||))) - e^-((1 + 2i + 3j + 4k)*((2i + 3j + 4k)/(||2i + 3j + 4k||)))) / (2*(1 + 2i + 3j + 4k)*((2i + 3j + 4k)/(||2i + 3j + 4k||))).

To find the cosine of a Quaternion in racket, type
> (cos (q 1 2 3 4))
(quaternion 58.93364616794397 -34.08618369046563 -51.12927553569845 -68.17236738093128)
The "(quaternion 58.93364616794397 -34.08618369046563 -51.12927553569845 -68.17236738093128)" that was returned is the cosine of the input quaternion.

2.1 Quaternion

struct

(struct quaternion (h i j k)
  #:extra-constructor-name make-quaternion)
  h : real?
  i : real?
  j : real?
  k : real?
A structure to represent a quaternion. h represents the real part, i, j and k represent the imaginary parts. If not provided with 4 real numbers, return a type-wrong-format error. Automatically defines (quaternion? x) to test if x is a quaternion. (quaternion-h x), (quaternion-i x), (quaternion-j x) and (quaternion-k x) can be used to get the components of a quaternion x.

procedure

(q h i j k) → quaternion?

  h : real?
  i : real?
  j : real?
  k : real?
Creates a quaternion. The exact same as (quaternion h i j k). Allows for faster typing.

procedure

(eq? x y) → quaternion?/number?

  x : quaternion?/number?
  y : quaternion?/number?
Defined in the module as (quaternion-equal number1 number2). Returns #t if the two arguments are equivalent mathematically, otherwise returns #f. If none of the arguments are quaternions, uses racket’s default eq? function, otherwise it converts each argument into a quaternion, then compares each corresponding element of two quaternions left to right. The and of those comparisons is returned.
> (eq? (quaternion 2 5 34 -1) (quaternion 2 5 34 -1))
#t
> (eq? (quaternion 2 5 34 -1) (quaternion -1 0 4 16))
#f

procedure

(quaternion->string x) → string?

  x : quaternion?/number?
Returns x as a string representation of a quaternion. Given in the form of "(quaternion h i j k)".
> (quaternion->string (quaternion 2 5 34 -1))
"(quaternion 2 5 34 -1)"

procedure

(make-quaternion x) → quaternion?

  x : quaternion?/number?
Only available within the module. Turns any number into a quaternion. If x is a quaternion, returns x. If x is a number, returns (quaternion (real-part x) (imag-part x) 0 0). If neither, returns (quaternion 0 0 0 0).
> (make-quaternion 2+3i)
(quaternion 2 3 0 0)

procedure

(unmake-quaternion x) → number?/quaternion?

  x : quaternion?
Only available within the module. If (quaternion-j x) and (quaternion-k x) are zero, returns a complex number that is mathematically equivalent to x, otherwise returns x.
> (unmake-quaternion (quaternion 2 3 0 0))
2+3i

procedure

(scalar-part x) → number?

  x : quaternion?/number?
Returns the real part of x as a real number. For quaternions it is equivalent to (quaternion-h x).
> (scalar-part (quaternion 2 5 34 -1))
2

procedure

(vector-part x) → quaternion?

  x : quaternion?/number?
Returns the imaginary part(s) of x as a quaternion with real part 0. It does this by converting the input to a quaternion, then subtracting the real part.
> (vector-part (quaternion 2 5 34 -1))
(quaternion 0 5 34 -1)

procedure

(norm x) → number?

  x : quaternion?
Defined within the module as (quaternion-norm number). Returns the norm of x. Also known as the magnitude, this represents the length of the quaternion when represented as a vector. Found by taking the square root of the sum of the squares of the components of x.
> (norm (quaternion 2 5 34 -1))
34.4383507154451252

procedure

(conjugate x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-conjugate number). Returns the conjugate of x. This is a new quaternion with the same real part as x and the opposite vector parts. The product of x and it’s reciprocal is the square of the norm. This a new quaternion with the same real part as x This is equivalent to (- (real-part x) (vector-part x)), however this function takes the opposite of each imaginary component individually. If none of the arguments are quaternions, uses racket’s default conjugate procedure.
> (conjugate (quaternion 2 5 34 -1))
(quaternion  2 -5 -34 1)

procedure

(reciprocal x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-reciprocal number). Returns the reciprocal of x. A reciprocal is defined such that the reciprocal of x times x is 1. This procedure returns a new quaternion whose elements are the corresponding elements of x is divided by the norm of x squared.
> (reciprocal (quaternion 2 5 34 -1))
(quaternion 0.0016863406408094439 -0.004215851602023609 -0.028667790893760547 0.0008431703204047219)

procedure

(unit w) → number?

  w : quaternion?
Defined within the module as (quaternion-unit number). Returns the unit quaternion of w. This is the quaternion divided by its norm, it represents the direction of the quaternion with a parallel quaternion of norm 1.
> (unit (quaternion 2 5 34 -1))
(quaternion 0.0016863406408094439 -0.004215851602023609 -0.028667790893760547 0.0008431703204047219)

2.2 Basic Operators

procedure

(+ x ...) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-add . quaternions). Returns the sum of all xs, or for only one x returns x. If none of the arguments are quaternions, uses racket’s default + procedure. Otherwise, converts all arguments to quaternions and returns a new quaternion whose components are the sums of the corresponding components of the arguments.
> (+ (quaternion 2 5 34 -1) (quaternion -1 0 4 16))
(quaternion 1 5 38 15)
> (+ (quaternion 1 2 3 4) 4 (quaternion 4 3 5 -3) 3-2i)
(quaternion 12 3 8 1)

procedure

(- x) → quaternion?/number?

  x : quaternion?/number?
(- x w ...) → quaternion?/number?
  x : quaternion?/number?
  w : quaternion?/number?
Defined within the module as (quaternion-subtract . quaternions). Returns the difference of w from x. If no ws are supplied, returns (- 0 x) For multiple ws takes the difference of each w from x. If none of the parameters are quaternions, uses racket’s default - procedure, otherwise makes each argument a quaternion, then returns a new quaternion whose elements are the correspoonding elements of x minus the corresponding element of each w.
> (- (quaternion 1 2 3 4))
(quaternion -1 -2 -3 -4)
> (- (quaternion 3 2 4 5) (quaternion 5 3 7 5))
(quaternion -2 -1 -3 0)
> (- (quaternion 10 6 22 -5) 4 (quaternion 2 3 4 1) 3+2i)
(quaternion 1 1 18 -6)

procedure

(quaternion-multiply2 x w) → quaternion?

  x : quaternion?
  w : quaternion?
Only defined within the module, used by the * and / procedures. Returns a quaternion product of two quaternions using the following formulae:
hnew = hx*hw - ix*iw - jx*jw - kx*kw
inew = hx*iw + ix*hw + jx*kw - kx*jw
jnew = hx*jw - ix*kw + jx*hw + kx*iw
knew = hx*kw + ix*jw - jx*iw + kx*hw
}

procedure

(* x ...) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-multiply . quaternions). Returns the product of all xs, or for only one x returns x. If none of the arguments are quaternions, uses racket’s default * procedure. Otherwise, converts all arguments to quaternions and uses quaternion-multiply2 to multiply all of the arguments pairwise from left to right.
> (* (quaternion 1 2 3 4) (quaternion 4 3 2 1))
(quaternion -12 6 24 12)
> (* (quaternion 1 2 3 4) 2 (quaternion 4 3 2 1) 4-8i)
(quaternion 0 240 0 480)

procedure

(/ x) → quaternion?/number?

  x : quaternion?/number?
(/ x w ...) → quaternion?/number?
  x : quaternion?/number?
  w : quaternion?/number?
Defined within the module as (quaternion-divide . quaternions). Returns x divided by w. If no ws are supplied returns (reciprocal x), which is equivalent to (/ 1 x). If none of the parameters are quaternions, uses racket’s default / procedure. Otherwise calls   x by the reciprocal of the ws pairwise.
> (/ (quaternion 1 1 1 1))
(quaternion .25 -.25 -.25 -.25)
> (/ (quaternion 1 2 3 4) (quaternion -4 2 1 3))
(quaternion 0.5 -0.5 -0.5 -0.5)
> (/ (quaternion 1 2 3 4) 1.5 (quaternion -4 2 1 3) 1+i)
(quaternion 0.1111111111111111 9.25185853854297e-018 -0.11111111111111106 -0.4444444444444443)

2.3 Advanced Operators

procedure

(exp x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-exp number). Returns Euler’s number raised to the power of x. If x is not a quaternion, or if the vector-part of x is the zero vector, uses racket’s default exp procedure. Otherwise, uses the equation ea * (cos(||v||) + unit(v) * sin(||v||)) where a is the scalar-part of x, v is the vector-part of x.
> (exp (quaternion 2 5 34 -1))
(quaternion 5.920249924190469 0.5765955512596606 4.38212618957342 -0.1153191102519321)

procedure

(log x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-log number). Returns the natural logarithm of x. If x is not a quaternion, or if the vector-part of x is the zero vector, uses racket’s default log procedure. Otherwise, uses the equation ln(||x||) + unit(v) * arccos(a/||x||) where a is the scalar-part of x, v is the vector-part of x.
> (log (quaternion 2 5 34 -1))
(quaternion 3.539170789778835 0.21999401685083464 1.4959593145856755 -0.04399880337016693)

procedure

(expt x y) → quaternion?/number?

  x : quaternion?/number?
  y : quaternion?/number?
Defined within the module as (quaternion-expt root power). Returns the x to the power of y. If if neither argument is a quaternion, uses racket’s default expt procedure. If y is 0, returns (quaternion 1 0 0 0). If x is 0 and y is not 0, returns (quaternion 0 0 0 0). In all other cases, uses the equation ey*log(x).
> (expt (quaternion 2 5 34 -1) (quaternion 1 2 3 4))
(quaternion -0.1742401274849861 0.01354098958151244 0.14979740084735807 0.18830957944329185)

procedure

(sin x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-sin q). Returns the sine of x. If x is not a quaternion, uses racket’s default sin procedure. Otherwise uses the equation (exV + e-xV)/2 where V is (unit (vector-part x)).
> (sin (quaternion 1 1 1 1))
(quaternion 2.4525323488837176 0.8539945649192702 0.8539945649192703 0.8539945649192702)

procedure

(cos x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-cos q). Returns the cosine of x. If x is not a quaternion, uses racket’s default sin procedure. Otherwise uses the equation (exV - e-xV)/(2 * V) where V is (unit (vector-part x)).
> (cos (quaternion 1 1 1 1))
(quaternion 1.5747529115583145 -1.3300177322185738 -1.3300177322185738 -1.3300177322185738)

procedure

(tan x) → quaternion?/number?

  x : quaternion?/number?
Defined within the module as (quaternion-tan q). Returns the tangent of x. If x is not a quaternion, uses racket’s default tan procedure. Otherwise uses the equation ((/ (sin q) (cos q)). Throws an error if (cos q) is 0.
> (tan (quaternion 1 1 1 1))
(quaternion 0.05838794357801061 0.5916176172069279 0.5916176172069277 0.5916176172069277)

3 qreader

Allows for the input of quaternions in the standard infix notation of a+bi+cj+dk.

To use qreader, require it, then simply type (start) in your interpreter. This will allow you to continue as normal using quaternions in infix notation. If you receive an error, you will have to start the qreader again.

procedure

(GoodExpression? s) → boolean?

  s : string?
Returns #t if the string is a valid quaternion in infix notaton.

procedure

(Expression->Quaternion s) → quaternion?

  s : string?
If (GoodExpression? s) then returns s as a quaternion struct, otherwise returns the error that syntax is not valid.

procedure

(mySymbol->String s) → string?

  s : symbol?/list?
Turns either a symbol or list into a string representing that symbol or list.

procedure

(splitToTokens s) → list?

  s : string?
returns the string as a list of strings of each token in s.

procedure

(replaceQuatToken Token) → string?

  Token : string?
Takes a string that is a single token. If that token matches the quaternion regex, returns it as a string representing the struct quaterniong i.e. "(quaternion 1 2 3 4)". Otherwise returns Token.

procedure

(replaceQuatString expString) → string?

  expString : string?
Goes through a string and replaces each instance of a quaternion in infix notation with the same quaternion in struct notation.

procedure

(start) → void?

Creates a interpreter environment in which quaternions can be typed in infix notation. Upon hiting enter, the line is executed and (start) is called again.
